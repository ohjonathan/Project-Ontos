### Dev 1 — Lock discipline + SF-8 + m-12 + A5 verify (PARALLEL with Dev 4)

**Primary references:** addendum §A1 + §A5, verdict SF-8 + m-12, C.0 Q2.

**Scope:**
- Add `owns_lock: bool = True` to `SessionContext.__init__()`.
- When `owns_lock=False`: `_acquire_lock()` and `_release_lock()` are no-ops; `commit()` assumes the caller holds the flock on `<workspace>/.ontos.lock`.
- Existing CLI callers pass nothing → default `owns_lock=True` → behavior unchanged.
- **m-12 fix** (`context.py:231-241`): close the file handle if an unexpected (non-`BlockingIOError`) exception is raised between `open()` and successful `flock`.
- **SF-8 test:** simulate a pre-v4.1 PID-based `.ontos.lock` file (arbitrary text content, no flock held) and confirm `_acquire_lock()` handles the encounter correctly.
- **A5 verify:** `.gitignore:22` already contains `.ontos.lock`. Confirm in PR description; no file modification needed.
- **Contention integration tests:**
  1. Two writers same workspace → second fails with `RuntimeError` (inter-process simulated via subprocess or `multiprocessing`).
  2. Writer vs `verify --portfolio` reader — reader should not block (read-tool path does not flock per C.0 Q2).
  3. **Regression test for A1:** `workspace_lock()` held + `SessionContext(owns_lock=False).commit()` — must succeed; the same test with default `owns_lock=True` must raise `RuntimeError` (or timeout).

**File targets:** `ontos/core/context.py` (primary); `tests/test_context.py` or new `tests/test_context_contention.py`.

**Closes:** SF-8, m-12, m-7 (substrate half), A1, A5.

### Dev 2 — Single-file write tools (depends on Dev 1)

**Primary references:** Implementation Spec v1.1 §4.8 (adjusted per addendum §A1, §A2, §A3), C.0 Q1 + Q3.

**Scope — three MCP tools:**

- `scaffold_document` — create a new `.md` in the target workspace.
- `log_session` — create `{logs_dir}/{date}_{slug}.md`.
- `promote_document` — mutate document frontmatter only.

**Cross-cutting requirements for all three:**

1. **Lock pattern (A1):**
   ```python
   with workspace_lock(cache.workspace_root):
       ctx = SessionContext(workspace_root=..., owns_lock=False)
       ctx.buffer_write(...)
       ctx.commit()
       portfolio_index.rebuild_workspace(slug, root)
   ```
2. **Slugifiers (A2):**
   - Workspace identity → `from ontos.mcp.scanner import slugify` (AND `allocate_slug` if collision handling needed).
   - Title → filename → `from ontos.commands.log import _slugify`.
   - **No third slugifier.**
3. **A3 post-commit failure handling:** if `ctx.commit()` raises, re-invoke `rebuild_workspace(slug, root)` in the exception path to converge the DB. Hook pattern mirrors `context.py:189-199`.
4. **`read_only` enforcement (closes m-2):** before any `buffer_write`, check workspace config's `read_only` flag; if true, return a structured MCP error (do not write). This is the first code path that consumes `read_only` — `_ = read_only` at `server.py:73,104` gets replaced.
5. **`workspace_id` validation (closes m-14):** route through one shared helper that all write tools call; close the redundant guards at `server.py:478` and `tools.py:513-525` (coordinate with Dev 3's m-8).
6. **FTS5 concurrency (CB-10 pattern):** any SQLite connection opened here uses `timeout=5.0`.
7. **Response models (closes m-15):** extend `TOOL_SUCCESS_MODELS` at `ontos/mcp/schemas.py:283` with Track B tool names.

**File targets:** `ontos/mcp/tools.py`, `ontos/mcp/server.py`, `ontos/mcp/schemas.py`, `tests/`.

**Closes:** m-2, m-7 (consumers), m-14, m-15 (partial).

**Branch:** `feature/v4.1-track-b-dev2` off `feature/v4.1-track-b-dev1`.

**PR target:** `feature/v4.1-track-b`.
