# Session Log: V2 Planning & Self-Development Setup
Date: 2025-12-12 09:57 KST
Source: Antigravity

## 1. Goal
Implement the **Ontos Self-Development Protocol** to allow Project Ontos to use its own tooling to document its development ("dogfooding"). This establishes the foundation for v2.0 development.

## 2. Key Decisions
-   **Hidden Directory:** Adopted `.ontos-internal/` for project documentation to avoid cluttering user projects.
-   **Smart Config:** Implemented conditional logic in `ontos_config.py` to auto-detect "Contributor Mode" (when `.ontos-internal` exists) vs "User Mode".
-   **Orphan Atoms:** Allowed `atom` type to be orphans in Contributor Mode to support standalone technical specs (`schema`, `architecture`).
-   **Dual Ontology:** Formally documented the separation of Space (Graph) and Time (Log) in `architecture.md`.

## 3. Changes Made
-   **Structure:** Created `.ontos-internal/` with `kernel`, `strategy`, `product`, `atom`, `logs` directories.
-   **Config:** Updated `ontos_config_defaults.py` and `ontos_config.py`.
-   **Documentation:**
    -   `kernel/mission.md`: Defined core mission.
    -   `strategy/v2_strategy.md`: Refined v2 strategy.
    -   `product/v2_roadmap.md`: Defined v2 phases.
    -   `atom/architecture.md`: Drafted v2 architecture (Space vs Time).
    -   `atom/schema.md`: Drafted frontmatter schema.
    -   `atom/self_development_protocol.md`: Codified development rules.
