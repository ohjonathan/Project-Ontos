## Findings

### [F1] CLI/MCP parity explicitly asserted in tests
- **Severity:** nit
- **Evidence:** static-inspection
- **Where:** `tests/commands/test_activate_json_warning_metadata.py:153-180`
- **Issue:** The mandatory parity test (`test_cli_and_mcp_serialization_identical_for_validation_result`) calls `_validation_issues(...)` directly to assert byte-equality with `[issue.to_dict() for issue in ...]`. This is exactly the load-bearing assertion B.1 sonnet F1 mandated. Note that `_validation_issues` is a private helper (leading underscore); importing it from `ontos.mcp.tools` in a test is acceptable because the test is testing the private surface deliberately. If `_validation_issues` is ever renamed or made truly internal, this test needs updating — but that should be tracked alongside such a rename, not pre-emptively here.
- **Recommendation:** No change needed. Note for future contributors: this test pins the implementation choice that `_validation_issues` delegates to `ValidationError.to_dict()`. If the MCP helper changes signature, update accordingly.
