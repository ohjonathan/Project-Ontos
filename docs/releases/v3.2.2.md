## Ontos v3.2.2

Patch release focused on shipping `ontos maintain` and aligning maintenance workflow documentation.

### Highlights
- Added native `ontos maintain` command to close the documentation/CLI gap.
- Weekly maintenance now runs 8 ordered tasks with error isolation.
- Updated maintenance docs across Agent Instructions and Manual.

### Added
- `ontos maintain` command registration and handler in the unified CLI.
- Registry-based maintenance runner in `ontos/commands/maintain.py`.
- New maintain test suite in `tests/commands/test_maintain.py`.
- Release notes source file at `docs/releases/v3.2.2.md`.

### Fixed
- Removed interactive no-op behavior in proposal review; now reports candidates for manual graduation.
- Dry-run behavior now short-circuits task scans where execution should not occur.
- AGENTS staleness checks now use the maintain command project root consistently.
- `MaintainContext.config` is now strongly typed as `OntosConfig`.

### Changed
- `ontos maintain` default task set:
  - `migrate_untagged`
  - `regenerate_map`
  - `health_check`
  - `curation_stats`
  - `consolidate_logs` (if `AUTO_CONSOLIDATE=True`)
  - `review_proposals`
  - `check_links`
  - `sync_agents` (if stale)
- `--skip` now documents all task names and supports comma-separated values.
- JSON output includes per-task `details`.
