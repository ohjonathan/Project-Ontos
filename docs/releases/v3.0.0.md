### Added

- **PyPI Distribution** â€” `pip install ontos` with console script entry point
- **Full argparse CLI** â€” 13 commands: `init`, `log`, `map`, `maintain`, `consolidate`, `query`, `update`, `doctor`, `hook`, `export`, `verify`, `agents`, `version`
- **`ontos agents`** â€” Generate `AGENTS.md` and `.cursorrules` for AI assistant integration
- **`ontos doctor`** â€” Health check with 8 validation checks (hooks, config, map freshness, agent staleness)
- **`ontos hook`** â€” Manage git hooks (`install`, `uninstall`, `status`, `validate`)
- **`ontos export`** â€” Export context for LLMs in JSON or Markdown format
- **JSON output support** â€” `--json` flag for machine-readable output on `map`, `doctor`, `export`, `query`
- **Security hardening** â€” Input validation, path traversal protection, secure file operations

### Changed

- **License** â€” Changed from MIT to Proprietary
- **Package structure** â€” `ontos/core/` (pure logic), `ontos/ui/` (I/O), `ontos/_scripts/` (backwards compat)
- **CLI entry point** â€” `ontos <command>` replaces `python3 ontos.py <command>`
- **Configuration** â€” Uses `pyproject.toml` or `ontos.toml` (TOML format), legacy `ontos_config.py` still supported

### Removed

- **Legacy scripts** â€” Direct `ontos_*.py` invocation deprecated
- **`ontos_lib.py`** â€” Fully removed (use `ontos.core.*` imports)

### Migration Guide

```bash
# Old way (deprecated)
python3 .ontos/scripts/ontos_end_session.py --source "Claude"

# New way
pip install ontos
ontos log --source "Claude"
```
