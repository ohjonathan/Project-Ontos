## What's Changed

**Major release: Ontos is now installable via `pip install ontos`.**

### 🎯 Highlights

- Full PyPI distribution — install with `pip install ontos`
- 13-command argparse CLI replacing legacy script invocation
- Proper Python package structure (`ontos/core/`, `ontos/ui/`)
- Security hardening with input validation and path traversal protection

---

### Added

- **PyPI Distribution** — `pip install ontos` with console script entry point
- **Full argparse CLI** — 13 commands: `init`, `log`, `map`, `maintain`, `consolidate`, `query`, `update`, `doctor`, `hook`, `export`, `verify`, `agents`, `version`
- **`ontos agents`** — Generate `AGENTS.md` and `.cursorrules` for AI assistant integration
- **`ontos doctor`** — Health check with 8 validation checks (hooks, config, map freshness, agent staleness)
- **`ontos hook`** — Manage git hooks (`install`, `uninstall`, `status`, `validate`)
- **`ontos export`** — Export context for LLMs in JSON or Markdown format
- **JSON output support** — `--json` flag for machine-readable output on `map`, `doctor`, `export`, `query`
- **Security hardening** — Input validation, path traversal protection, secure file operations

### Changed

- **License** — Changed from MIT to Proprietary
- **Package structure** — `ontos/core/` (pure logic), `ontos/ui/` (I/O), `ontos/_scripts/` (backwards compat)
- **CLI entry point** — `ontos <command>` replaces `python3 ontos.py <command>`
- **Configuration** — Uses `pyproject.toml` or `ontos.toml` (TOML format), legacy `ontos_config.py` still supported

### Removed

- **Legacy scripts** — Direct `ontos_*.py` invocation deprecated
- **`ontos_lib.py`** — Fully removed (use `ontos.core.*` imports)

### Migration Guide

```bash
# Old way (deprecated)
python3 .ontos/scripts/ontos_end_session.py --source "Claude"

# New way
pip install ontos
ontos log --source "Claude"
```
