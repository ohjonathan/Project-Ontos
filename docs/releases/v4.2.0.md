# v4.2.0 — "Cursor Onboarding + Universal Fallback"

**Release date:** 2026-04-14

Patch release narrowing Ontos MCP onboarding to the next managed client
that has a simple native contract: Cursor.

---

## Summary

- `v4.1.3` fixed the Antigravity-native onboarding gap.
- `v4.2.0` keeps Antigravity working, adds Cursor as the only new
  first-class managed client, and adds `ontos mcp print-config` as the
  universal fallback for the remaining supported client surfaces.
- `v4.2.0` does **not** add a standalone `refresh` subcommand. Rerunning
  `ontos mcp install --client ...` is the supported refresh path.

## Fixes

### Cursor managed onboarding

Ontos now ships:

```bash
ontos mcp install --client cursor --scope project
ontos mcp install --client cursor --scope user
ontos mcp uninstall --client cursor --scope project
```

Cursor project scope uses `.cursor/mcp.json`; user scope uses
`~/.cursor/mcp.json`.

The installer preserves unrelated `mcpServers` entries, compares only
Ontos-owned `command` and `args` for `noop`, and rewrites the launcher
path when rerun after Ontos moves on your shell `PATH`. The `noop`
result is a behavior clarification, not a breaking change: it means the
managed entry already matches the intended Ontos-owned fields, so the
rerun does not rewrite the file.

### Universal fallback

Ontos now ships:

```bash
ontos mcp print-config --client codex
```

`print-config` is the zero-write fallback for Antigravity, Cursor,
Claude Code, Codex, and VS Code. It emits a complete, copy-pastable
document instead of mutating the target config.

### Doctor coverage

`ontos doctor` now includes `cursor_mcp`, with project-precedence
aggregation and malformed-file warnings that match the spec. Existing
Antigravity doctor coverage remains intact.

### Platform guidance

Managed MCP automation in `v4.2.0` is POSIX-only. Windows users should
use `print-config` instead of managed install / uninstall / doctor
commands.

### Contract drift

If a managed client contract drifts from the documented source of truth,
Ontos will surface the failure clearly and target either a docs/snippet
fix or an explicit incompatibility message within 72 hours of a
reproduced report.

## Docs Updated

README, manual, migration guidance, and this release note now separate:

- repo-local instruction artifacts such as `AGENTS.md` and `.cursorrules`
- native MCP client configuration
- managed clients vs `print-config`-only clients

## Acceptance

- `ontos mcp install --client cursor --scope project` creates a valid
  native Cursor config.
- `ontos mcp uninstall --client cursor --scope project` removes only the
  Ontos entry.
- `ontos doctor` reports Cursor config state with actionable remediation.
- `ontos mcp print-config --client codex` emits a full TOML document.
- Windows users still have a supported path via `print-config`.
