# v4.3.0 — "Obsidian Write Path"

**Release date:** 2026-04-15

Minor release shipping `ontos retrofit --obsidian`, a scoped bulk-apply command that writes computed `tags:` and `aliases:` frontmatter fields into existing documents so Obsidian vaults can browse Ontos-managed docs without hand-editing every block.

---

## Highlights

- **`ontos retrofit --obsidian`** — Writes computed `tags` (from `concepts`) and `aliases` (explicit + auto-generated from `id`) fields into on-disk frontmatter. Dry-run by default; `--apply` requires clean git.
- **Shared frontmatter-edit helpers** — Extracted `_read_decoded_content`, `_split_frontmatter`, `_index_top_level_fields`, and `_check_clean_git_state` from `rename.py` into `ontos/core/frontmatter_edit.py` so `retrofit` and `rename` share the same line-indexing logic.

---

## Features

### `ontos retrofit --obsidian`

The read path for Obsidian compatibility shipped in v3.1.0 — `normalize_tags()` and `normalize_aliases()` have been populating `DocumentData.tags` / `DocumentData.aliases` in memory since then. v4.3.0 adds the **write path**: a scoped command that diffs canonical values against on-disk frontmatter and inserts, replaces, or removes blocks surgically.

Usage:

```bash
ontos retrofit --obsidian                      # dry-run (default)
ontos retrofit --obsidian --apply              # write frontmatter
ontos retrofit --obsidian --scope library      # include .ontos-internal
```

Per-field behavior:

- **Absent** → inserts a block-style list right before the closing `---`.
- **Present and matches computed** → no-op; file untouched.
- **Present but drifted** (e.g. `tags` out of sync with `concepts`) → replaces the field's line range with the canonical sorted list.
- **Present but canonical value is empty** → removes the stale field block.

---

## Safety

- **Dry-run by default.** `--apply` is required to write files.
- **Clean git required for `--apply`.** Matches `ontos rename` behavior.
- **Atomic commit.** Writes route through `SessionContext.buffer_write` + `commit()` — temp files staged, then renamed, with flock serialization.
- **Unpatchable frontmatter blocks apply for the batch.** Anchors, aliases, merge-derived target values, YAML tag syntax, block scalars, and duplicate top-level keys raise blocking warnings and abort `--apply`.
- **Post-apply reminder.** Output notes: `Run 'ontos map' to regenerate derived artifacts.`
