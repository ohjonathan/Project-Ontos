# v3.4.0 — "Tiered Context"

**Release date:** 2026-04-04

Minor release shipping `--compact tiered` context maps — a prose summary plus type-ranked compact output for token-constrained agents.

---

## Highlights

- **`--compact tiered` mode** — `ontos map --compact tiered` produces three-section output: prose project summary (Tier 1), type-ranked compact listing (Tier 2), and full ID index (Tier 3).
- **Two review rounds of hardening** — Sort key consistency, log contract enforcement, project_root normalization, and crash fixes from PR #79 review cycles.

---

## Features

### `--compact tiered` Mode (PR [#79](https://github.com/ohjonathan/Project-Ontos/pull/79))

New compact output format designed for token-constrained agents that need structured context without the full graph:

- **Tier 1:** Prose project summary with recent activity and key documents
- **Tier 2:** Type-ranked compact listing with optional summaries
- **Tier 3:** Full document ID index

Usage:
```bash
ontos map --compact tiered
```

Joins existing `--compact` (minimal one-line-per-doc) and `--compact rich` (with summaries) modes.

---

## Bug Fixes

- Shared `_sort_key` across all compact modes for deterministic output.
- Log contract enforcement in tiered summary generation.
- Reverted anti-scope violations in `_generate_tier1_summary()`.
- Final tiered review debt removed (spec cleanup).
