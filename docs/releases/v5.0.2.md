# Ontos v5.0.2

v5.0.2 is a patch release that preserves the v5 CLI and package contracts. It
hardens the exact-artifact TestPyPI gate against the upload-to-Simple-API
propagation delay observed during v5.0.1, and completes two inherited test
hygiene findings tracked by #148.

## Provenance-gate propagation handling

The TestPyPI JSON gate still verifies the exact wheel and sdist filenames,
sizes, kinds, and SHA-256 hashes before download. The subsequent hash-locked
pip download now polls a bounded 12 times at 10-second intervals when—and only
when—pip reports that `ontos==<tag version>` is not yet available from the
TestPyPI Simple API.

Hash mismatches are never retried. Neither are unrelated pip errors or a wheel
whose bytes, filename, metadata, or version differ from the immutable release
manifest. Those conditions fail immediately and continue to block production
publication.

## Test hygiene

The unused `legacy` pytest marker is removed. Repetitive subprocess-spawned
command help tests are replaced by a recursive in-process assertion that the
argparse tree exactly matches the declarative command registry, every parser
can render help, visibility follows registry metadata, and selected command
help remains pinned to golden text. Subprocess tests remain where the process
boundary is the behavior under test: `python -m ontos --help`, arbitrary-CWD
imports, and MCP stdout safety.

Release tags, publication, GitHub release creation, and #148 closure remain
maintainer-owned actions and are not claimed by this source change.
