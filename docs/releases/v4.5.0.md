# Ontos v4.5.0

v4.5.0 closes three open GitHub issues against `ohjonathan/Project-Ontos` in one strict-P3 deliverable: schema-safe MCP `get_context_bundle` warnings, activation/link-check diagnostic signal hardening, and MCP host reload documentation.

## Bug fixes

- **`get_context_bundle` MCP schema rejection (#115).** Calling `get_context_bundle` before `activate` previously injected an undeclared `_ontos_warning` key into the response, which the MCP SDK rejected against the strict `additionalProperties: false` output schema. The pre-activate reminder now lands in the already-declared `warnings: List[str]` field for `get_context_bundle`; legacy `READ_WARNING_TOOL_NAMES` tools (e.g., `workspace_overview`, `context_map`) continue to receive the `_ontos_warning` property whose schema advertises that key.

## Activation / link-check signal hardening (#117)

- **`depends_on` path-resolution fallback.** `build_graph` now accepts an optional `workspace_root`. When provided, `depends_on:` entries that don't match a loaded doc id are tried as workspace-relative, declaring-doc-relative, then absolute paths before being reported as broken. Path-matches that point at a loaded doc become normal edges (closes the 10 false-positive errors observed against `company-os`); existing-but-not-loaded paths become a soft `OUT_OF_SCOPE_DEPENDENCY` warning instead of a hard error.
- **Warning enrichment.** Orphan, depth, log-field, broken-link, and out-of-scope warnings now carry `rule_id`, `document_id`, and `file_path` in the MCP activation payload — agents and humans can triage without a second query. Snapshot bare-string warnings are classified into rule ids via prefix matching.
- **Widened `DocumentType` / `DocumentStatus` vocabulary.** Lifecycle artifact types — `handoff`, `tracker`, `retro`, `review`, `spec`, `report`, `adr`, `policy` — and lifecycle workflow statuses — `proposed`, `ready`, `completed`, `revised`, `in-lifecycle` — are now first-class canonical values. Real engineering workspaces stop falling 50% into `unknown`. Conservative repair still falls back to `unknown` when the widened vocabulary cannot match.
- **`body.bare_id_token` tightening.** Generic-mode link-check no longer fires on every prose token containing `_`, `.`, or a digit. The reporter measured 11,163 false positives across a 163-doc corpus; this release narrows generic mode to require an explicit `[[id]]` wikilink sigil. Known-ID detection, markdown link target detection, and rename mode are unchanged.
- **README / template skip.** `README.md` and `*_template.md` files are no longer loaded as data docs unless they declare an explicit `id:` frontmatter field — closing the "Log missing fields: branch" warning on `docs/logs/README.md`.
- **`ontos doctor` severity alignment.** Doctor now runs the same snapshot + validation pipeline as `ontos activate` and contributes a `failed` status (and non-zero exit) when activation reports any error-severity entry. Warnings alone remain `warning` (today's behavior).

## Documentation (#116)

- **Restart MCP hosts after upgrading Ontos.** Added a "Restart MCP hosts after upgrading" section to `README.md`, `docs/reference/Ontos_Manual.md`, and `docs/reference/Migration_v3_to_v4.md`. Long-lived MCP hosts (Claude Code, Cursor, Antigravity) spawn `ontos serve` once and keep that child process alive across `pipx upgrade ontos`, `pip install --upgrade ontos`, and `pipx install --force 'ontos[mcp]'`. Restart the MCP host (or reload the Ontos plugin) so it picks up the new version. The CLI shows the new version immediately because each CLI invocation is a fresh process.

## Test coverage
