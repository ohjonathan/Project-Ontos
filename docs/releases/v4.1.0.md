# v4.1.0 ‚Äî "Portfolio Authority"

**Release date:** 2026-04-12

Major release shipping portfolio-scoped MCP with read AND write tools, a portfolio index with FTS5 search, advisory flock locking for concurrent writes, wired bundle configuration, and a unified rename-plan orchestrator shared by CLI and MCP.

---

## Highlights

- **4 new MCP write tools** ‚Äî `scaffold_document`, `log_session`, `promote_document`, `rename_document`. All respect `read_only` at registration time, validate `workspace_id` via a shared helper, and use the `owns_lock=False` `SessionContext` pattern inside `workspace_lock()` for safe nesting.
- **3 new MCP read tools + `verify --portfolio`** (landed in Track A, PR #85) ‚Äî `project_registry`, `search_portfolio`, `get_context_bundle` over a per-session SQLite portfolio index with FTS5 full-text search.
- **Single-lock discipline** ‚Äî `workspace_lock()` advisory flock on `<workspace>/.ontos.lock` is the one mutual-exclusion primitive for Ontos write paths. `SessionContext` now supports `owns_lock=False` so nested callers inside `workspace_lock()` don't deadlock.
- **Bundle config wired through** ‚Äî `bundle_token_budget`, `bundle_max_logs`, `bundle_log_window_days` in `~/.config/ontos/portfolio.toml` now actually reach `build_context_bundle()`. New defaults: `8000` / `20` / `30`.
- **Shared rename orchestrator** ‚Äî `build_rename_plan` in `ontos/commands/rename.py` is the single plan-builder used by both `ontos rename --apply` and MCP `rename_document`. No more parallel CLI-vs-MCP plan loops.
- **Rollback-then-rebuild recovery** ‚Äî all write tools now revert partial file state on commit failure BEFORE rebuilding the portfolio index, preventing the DB from pointing at corrupted filesystem content.

---

## New MCP Tools

### Read-side (Track A, PR [#85](https://github.com/ohjonathan/Project-Ontos/pull/85))

- `project_registry` ‚Äî inventory of all known workspaces.
- `search_portfolio` ‚Äî FTS5 full-text search across workspaces with optional `workspace_id` scope.
- `get_context_bundle` ‚Äî token-budgeted bundle honoring the full portfolio config.

### Write-side (Track B, PR [#92](https://github.com/ohjonathan/Project-Ontos/pull/92))

- `scaffold_document` ‚Äî create a new markdown file in a workspace.
- `log_session` ‚Äî create a dated session log at `{logs_dir}/{date}_{slug}.md`.
- `promote_document` ‚Äî mutate frontmatter curation level only; no content move.
- `rename_document` ‚Äî rename an ID across all N referencing files in one transactional pass. LIBRARY scope unconditionally.

All four write tools:

- Refused when server is `--read-only` (tools not registered at all, not just refused at call time).
- Accept optional `workspace_id` ‚Äî falls back to the served workspace; explicit values validated.
- Wrap the entire mutation in `workspace_lock()` for cross-process mutual exclusion.
- On commit failure: scoped rollback (tracked paths `git checkout -- <path>`, untracked paths `unlink`), then portfolio-index rebuild, then re-raise the original exception.

## Verify subcommand

`ontos verify --portfolio` (landed in Track A) audits portfolio consistency:

- Slug-collision detection via the shared `allocate_slug` helper.
- Shared registry parser (`scanner.load_registry_records`) handles top-level list, `projects`, `workspaces`, `entries`, `items`, and dict-shaped registries.
- Malformed TOML produces a clean exit 2 with structured error, not a traceback.

## Configuration

### Portfolio config (`~/.config/ontos/portfolio.toml`)

```toml
[portfolio]
scan_roots = ["~/Dev"]
exclude = ["~/Dev/.dev-hub", "~/Dev/archive"]
registry_path = "~/Dev/.dev-hub/registry/projects.json"

[bundle]
token_budget = 8000
max_logs = 20
log_window_days = 30
```

`token_budget` is a hard cap with tail-first truncation. `max_logs` and `log_window_days` apply to logs independent of the budget.

### Server flags

- `--read-only` ‚Äî starts an MCP server that registers only read and portfolio tools. Write tools are absent from `list_tools()` output.
- `--port` and `--host` unchanged from v4.0.

## Breaking Changes

None for existing Ontos users. All v4.0 MCP tools preserve their signatures (added-only `workspace_id: str | None = None` parameter). CLI commands unchanged.

**For `ontos serve` consumers using `read_only=True`:** write tools are no longer advertised via `list_tools()` ‚Äî clients that previously tried to invoke them and got `E_READ_ONLY` will now get `Unknown tool: <name>`. This is the correct behavior but clients should update expectations.

---

## Upgrade Notes

- **Python 3.9+ still supported.** Despite a brief CI drift, base package installs cleanly on 3.9.
- **`.ontos.lock` appears in workspace roots** during writes. Confirm your `.gitignore` includes `.ontos.lock` ‚Äî the repo template already does.
- **`load_portfolio_config()` now raises `FileNotFoundError`** if `~/.config/ontos/portfolio.toml` is absent. Call `ensure_portfolio_config()` to initialize on first run (done automatically by the CLI entrypoints).
