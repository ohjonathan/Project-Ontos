# Ontos v4.4.0

v4.4.0 focuses on agentic activation resilience:

- `ontos activate` provides best-effort activation status and JSON diagnostics.
- MCP adds `activate`, per-session activation state, and skipped-activation warnings.
- MCP adds `session_end` for structured session archive logs.
- Frontmatter enum diagnostics now report path, line, field, observed value, allowed values, severity, blocking status, and suggested fixes.
- `ontos doctor --frontmatter` and `ontos maintain --fix-frontmatter-enums` support diagnosis and conservative repair.
- Generated context maps now use valid `status: complete` frontmatter while preserving `generated_by: ontos map`.
- `map`, `doctor`, `verify --all`, and repair workflows share configured scan exclusions.
- `doctor` recognizes linked git worktrees through `git rev-parse --is-inside-work-tree`.
